# RAG 检索式搜索 —— 需求文档

> 状态：**已确认，待排期实施**。基于 2026-07-16 与用户的逐轮对齐结论整理。
> 相对 [`docs/2026-07-12-rag-requirement.md`](../../2026-07-12-rag-requirement.md) 的演进见「与 07-12 版本的差异」。
> 配套技术方案见 [`tech-design.md`](./tech-design.md)。

## 一句话目标

在 ToucanShelf 内提供**检索式搜索**（全文关键词 + 向量语义的混合检索），让用户在几百篇文档的知识库里，
通过一次搜索快速定位到相关文档并跳转打开。**本期不做 LLM 生成回答**，只做「搜索 → 命中列表 → 跳转」。

## 与 07-12 版本的差异（重要）

07-12 的需求/技术文档设想的是 **RAG 问答**（检索 top-K → 喂 LLM → 生成回答），并明确「不做多租户
权限隔离」。本期方向调整如下：

| 维度 | 07-12 版本 | 本期（07-16） |
|---|---|---|
| 交付形态 | 检索 + LLM 生成回答（SearchChat） | **仅检索**，命中列表 + 跳转，无生成 |
| 检索方式 | 纯向量语义 | **混合检索**：FTS 全文分词 + 向量语义 + RRF 融合 |
| 向量存储 | 优先 sqlite-vec | **放弃 sqlite-vec**（纯 Go 驱动不支持），改 BLOB + Go 内存 cosine |
| 权限 | 明确不做 | **做权限收敛**：只搜用户有权访问的内容 |
| 索引更新 | 手动全量 | **异步增量 + 手动全量重建** |

## 架构约束（用户确认）

1. 数据库今后仅支持 **PostgreSQL 与 SQLite**，停止对其他数据库的支持。
2. 本期**只实现 SQLite 方案**。PG 的 pgvector 方案列入未来开发计划，本期不做。
3. 采用标准 RAG 检索流程：**向量搜索 + chunk 分片 + 全文分词检索**。
4. SQLite 方案定位于**个人和小团队**，**不考虑海量数据与高并发**，只做最低成本支持。

## 功能需求

### F1. 全局搜索（左侧主导航）

- 左侧主导航新增「搜索」入口，点击进入**独立的全局搜索页面**（占主区域，含输入框 + 命中列表）。
- **回车/点按钮提交**后才检索（非输入即搜，避免频繁触发 embedding API 烧 token）。
- 提交后展示**命中列表**。
- 点击命中项 → 跳转首页，定位到对应的**知识库与文档**并打开。
- 检索范围 = **当前用户有权访问的全部知识库**（自己的 + 有权限的），不会搜到他人私有内容。

### F2. 知识库内搜索（首页次级侧栏）

- 首页左侧次级侧栏，在 **Calendar 下方**新增搜索框，支持**当前知识库内**搜索。
- **提交搜索**后：
  - 次级侧栏的**层级文件夹区**过滤掉未命中的文档，**只保留命中文档及其父级文件夹路径**（保留层级上下文）。
  - **预览区复用命中列表**呈现。
- 点击命中项 → 跳转到对应文档。
- **清空搜索框** → 文件夹树恢复完整，预览区恢复原样。
- 检索范围 = 当前所在的**单个知识库（workspace）**。

### F3. 检索行为

- 混合检索：FTS 全文分词 + 向量语义，用 **RRF** 融合排序。
- 结果**文档级去重**：一个文档命中多个 chunk → 合并成一条，取最高分，展示最佳片段作为 snippet。
- 命中列表每项 = 一个文档 + 一段高亮摘要（snippet）。
- 点击命中项**只打开文档**，本期不做片段内定位/滚动高亮（留作未来增强）。

### F4. 索引范围

- 本期**仅索引普通 Markdown memo**。
- HTML / PDF（仅渲染类型）、`.view`（配置类）本期**不索引**。
- 架构上把「哪些文档进索引」设计成**可扩展过滤器**；未来在**知识库详情**页做成可配置项。

### F5. 降级策略

- embedding 依赖 AI provider 的 API key。当**未配置 embedding 模型**时：
  - 搜索**自动降级为纯 FTS 关键词检索**，保证功能始终可用。
  - 前端提示用户「配置 embedding 模型以启用语义搜索」。

### F6. Settings / Preferences 配置

- **instance 级**（全局一份）：embedding 模型配置——复用现有 AI provider 配置，在其中**新增
  「embedding 模型」字段**，并提示用户配置。
- **user 级**（每人可不同）：
  - **最大命中文档数**（返回命中列表的上限，默认建议 20）。
  - **检索模式**：`混合`（默认）/ `仅关键词` / `仅语义`；未配 embedding 时锁定为「仅关键词」。

## 明确不做（本期）

- LLM 生成式回答（问答/聊天）。
- PG / pgvector（未来阶段）。
- sqlite-vec、外部向量数据库、独立检索微服务。
- HTML / PDF / `.view` 文档的索引。
- 命中片段的精确定位与滚动高亮。
- 分片大小 / overlap 暴露为用户配置（写死合理默认）。

## 验收

- 未配 embedding：纯 FTS 可用，中文能按词/子串命中。
- 配置 embedding 后：混合检索生效，语义相关文档能被召回。
- 全局搜索：跨知识库、遵守权限边界；库内搜索：限定 workspace，文件夹树正确过滤/恢复。
- 换 embedding 模型：触发后台重建，旧向量作废、新向量生效。
- 用 10–20 个真实问题人工评估命中质量（沿用 07-12 的评估思路）。
